## 🔮 1단계: 데이터 준비 및 전처리

가장 기본이지만 모델의 성패를 좌우하는 단계입니다.

- **입력 변수 (Input, $X$):** 섬유 방사 공정 변수(예: 방사 온도, 압력, 용매 농도, 권취 속도 등)가 명확히 정의되었나요?
    
- **출력 변수 (Output, $Y$):** 최적화할 다중 목표 물성(예: 인장 강도, 신율, 생분해 속도, 직경 균일도 등)이 명확히 정의되었나요?
    
- **데이터 품질 및 양:**
    
    - 모델링(특히 상호 의존성 학습)에 **충분한 양**의 데이터가 확보되었나요? 데이터가 적으면 RC 모델이 실제 물리적 관계를 학습하기 어렵습니다.
        
    - 데이터에 결측치나 이상치(Outlier)는 없나요? 이는 모델 성능을 심각하게 저해할 수 있습니다.
        
- **데이터 스케일링:** 입력(공정 변수)과 출력(물성)은 각기 다른 단위와 범위를 가집니다. RC 모델 학습과 GA 탐색 효율을 위해 **정규화(Normalization) 또는 표준화(Standardization)**가 필수입니다.

## ⛓️ 2단계: 회귀 체인 (RC) 모델링

이 파이프라인의 '엔진'입니다. 이 모델이 부정확하면 GA는 엉뚱한 답을 찾게 됩니다.

- **🎯 목표 변수 순서 (Target Ordering):**
    
    - **가장 중요한 고려사항입니다.** RC는 $Y_1 \rightarrow Y_2 \rightarrow \dots \rightarrow Y_k$ 순서로 의존성을 학습합니다. ($Y_2$ 예측 시 $Y_1$을 입력으로 사용)
        
    - 이 순서를 어떻게 정할 것인가요?
        
        - **도메인 지식:** 물리적으로 가장 기본이 되는 물성(예: 결정성)부터 파생되는 물성(예: 강도) 순으로 정하는 것이 가장 좋습니다.
            
        - **통계적 접근:** 도메인 지식이 없다면, 변수 간 상관관계 분석(Correlation Analysis)을 통해 순서를 정하거나, 여러 순서 조합을 테스트(앙상블)하는 것을 고려해야 합니다.
            
- **기반 회귀 모델 (Base Regressor):**
    
    - 체인의 각 단계($Y_i$ 예측)에서 어떤 모델을 사용할 것인가요? (예: Random Forest, Gradient Boosting (XGBoost, LightGBM), SVM, 신경망 등)
        
    - 섬유 방사 공정은 비선형성이 강할 가능성이 높으므로, **비선형 모델(예: XGBoost, Random Forest)** 사용을 권장합니다.
        
- **모델 성능 검증:**
    
    - 각 출력 변수 $Y_i$에 대해 $R^2$ (결정계수), RMSE (평균 제곱근 오차) 등으로 모델의 정확도를 **개별적이고 철저하게** 검증해야 합니다.
        
    - **RC의 핵심:** 단순히 $k$개의 독립된 모델을 학습시킨 것보다 RC를 사용했을 때 **예측 성능이 실제로 향상되었는지** 비교 검증해야 합니다. (의존성 학습이 유의미했는지 확인)

## 🧬 3단계: 유전 알고리즘 (GA) 설계

GA는 RC 모델이라는 '지도'를 보고 최적의 '길'(공정 변수)을 찾는 '탐험가'입니다.

- **알고리즘 선택:**
    
    - "다중 목표"와 "파레토 프론트"를 언급하셨으므로, 단순 GA가 아닌 **다중 목표 유전 알고리즘(MOGA)**을 사용해야 합니다.
        
    - 가장 표준적이고 강력한 알고리즘은 **NSGA-II** (Non-dominated Sorting Genetic Algorithm II)입니다. SPEA2 등 다른 알고리즘도 고려할 수 있습니다.
        
- **염색체 인코딩 (Encoding):**
    
    - 입력 변수(공정 변수)를 어떻게 유전자로 표현할 것인가요? 공정 변수가 연속적인 값이므로 **실수 인코딩(Real-valued Encoding)**이 적합합니다.
        
- **탐색 공간 제약 (Constraints):**
    
    - 실제 공정에서 불가능한 변수 조합(예: 온도 $100^\circ\text{C}$ 미만, 압력 $50\text{bar}$ 초과)이 있습니다. 이 **제약 조건(Boundary Constraints)**을 GA 탐색 시 명확히 설정해야 합니다.
        
- **GA 주요 파라미터:**
    
    - **인구 집단 크기 (Population Size):** 너무 작으면 지역 최적해(Local Optima)에 빠지고, 너무 크면 계산 비용이 급증합니다.
        
    - **교차 (Crossover) 및 돌연변이 (Mutation) 방식:** 실수 인코딩에 적합한 방식(예: SBX Crossover, Polynomial Mutation)을 선택해야 합니다.
        
    - **종료 조건:** 최대 세대 수(Max Generations) 또는 파레토 프론트가 더 이상 개선되지 않을 때(Stagnation)로 설정합니다.

## 🎯 4단계: 적합도 함수 (Fitness Function) 정의

정확히는 MOGA의 '목적 함수(Objective Function)' 정의입니다.

- **RC 모델의 역할:** GA가 특정 공정 변수 조합($X_{\text{candidate}}$)을 제안하면, 훈련된 RC 모델이 이 입력을 받아 **예상되는 다중 물성 값($Y_{\text{pred}}$)을 반환**합니다. 이 $Y_{\text{pred}}$ 벡터가 MOGA가 평가할 대상입니다.
    
- **목표 설정 (Objective Definition):**
    
    - "다중 목표를 동시에 만족한다"는 것은 구체적으로 무엇인가요?
        
        - **특정 목표값 (Target) 근접:** (예: 강도=$T_1$, 신율=$T_2$)
            
            - 이 경우 목적 함수는 **'오차의 최소화'**가 됩니다.
                
            - $\text{Minimize } [ |Y_{\text{pred}, 1} - T_1|, |Y_{\text{pred}, 2} - T_2|, \dots, |Y_{\text{pred}, k} - T_k| ]$
                
        - **특정 방향 (Direction) 최적화:** (예: 강도는 최대화, 분해 속도는 최소화)
            
            - $\text{Minimize } [ -1 \times Y_{\text{pred}, 1} \text{ (강도)}, Y_{\text{pred}, 2} \text{ (분해 속도)}, \dots ]$ (GA는 보통 최소화 문제로 통일합니다)
                
- **적합도 평가 방식:** (NSGA-II 기준)
    
    - GA는 이 $Y_{\text{pred}}$ 벡터들을 **비지배 정렬(Non-dominated Sorting)**하여 순위(Rank)를 매깁니다.
        
    - 같은 순위 내에서는 **밀집도 거리(Crowding Distance)**를 계산하여 다양한 해를 유지하도록 유도합니다. (이 과정을 통해 파레토 프론트가 자연스럽게 도출됩니다.)

## 📈 5단계: 파레토 프론트 도출 및 최종 검증

최적화의 결과물이자 가장 중요한 실용적 단계입니다.

- **파레토 프론트 해석:**
    
    - GA가 종료되면 하나의 '최적해'가 아닌, **'최적해 집합'(파레토 프론트)이 도출**됩니다.
        
    - 이 집합 위의 모든 해는 "다른 물성을 희생하지 않고서는 한 물성을 더 개선할 수 없는" 해입니다. (Trade-off 관계)
        
- **후보 해 선택 (Decision Making):**
    
    - 도출된 파레토 프론트를 시각화하여(2~3개 목표 시) 엔지니어가 **실제 공정 조건, 비용, 중요도** 등을 고려하여 최종 운영 지점(Operating Point)을 선택해야 합니다.
        
- **🚨 최종 실물 검증 (Real-World Validation):**
    
    - **가장 중요합니다.** GA가 찾은 최적해($X_{\text{opt}}$)는 **RC 모델 상의 최적해**이지, **실제 공정의 최적해가 아닐 수 있습니다.**
